# Final Project Submission Checklist

This checklist summarizes the course rubric and the final paper/code submission instructions shown in `project_rubric.jpg` and `final_project_paper_and_code.jpg`.

## Paper Format

- Use the IEEE Conference template.
- Submit the paper as a PDF.
- Keep the paper to a maximum of 8 pages, excluding references.
- Follow the approved project proposal.
- Include exploratory data analysis and visualizations.
- Include, at minimum: title, authors, abstract, introduction, previous work, methodology, results, conclusions, and references.

## Rubric Priorities

| Section | Weight | Proficient-level target |
|---|---:|---|
| Abstract | 12% | One concise paragraph, no more than 150 words, with the problem, work completed, and key findings. |
| Introduction | 10% | Clear problem and contribution; no more than 20% of the paper. |
| Previous Work | 13% | At least 10 findable references, a concise state-of-the-art discussion, and a clear gap; no more than 30% of the paper. |
| Methodology | 15% | Clear description of data, languages and libraries, analysis setup, metrics, and why they were selected; normally 10-30% of the paper. |
| Results | 20% | Concise presentation of all key data using figures and tables, with every important visual referred to and discussed in the text. |
| Conclusions | 10% | Findings supported by the Results section, with limitations and future work. |
| Grammar and spelling | 10% | Easy to read, with few or no language errors. |
| Innovation | 10% | A clear and successful attempt to extend known approaches with a useful idea or perspective. |

## Code Package

- Submit code as a ZIP file.
- Include all source code.
- Include a README explaining setup and run steps.
- Include a small sample of the data that allows the code to run; do not include only the full dataset.
